GPT-4o 正式走入歷史。不是悄悄降級到遺留層，不是緩慢淡出——而是徹底退場。2026 年 4 月 3 日，OpenAI 完成了史上部署最廣泛的 AI 模型之一的全面退役，連同 GPT-4.1、GPT-4.1 mini 和 o4-mini 一併下架。這波分階段退役從 2 月 13 日啟動；到退役完成時，每日活躍 ChatGPT 用戶中，主動選擇 GPT-4o 的比例僅剩 0.1%。

這個數字，是整場過渡中最能說明問題的一個數據點。GPT-4o 於 2024 年 5 月問世時，是貨真價實的里程碑——比 GPT-4 Turbo 更快、更便宜，且能力強大到讓人感受到質的飛躍。它成為數億用戶的預設模型，也是數千個應用程式的基礎。兩年後，每一千名用戶中只有一人還在選擇它，清楚說明了技術前沿移動的速度。

## GPT-5.x 的架構

截至 2026 年 4 月，OpenAI 的現有模型陣容以 GPT-5.x 系列為核心，該系列在 2025 年底至 2026 年初分三個階段推出。GPT-5.2 是標準基準版本——大多數 ChatGPT 用戶日常互動的模型，也是 OpenAI 援引以支持退役 GPT-4o 的理由（內部說法是「GPT-5.2 已經足夠好」）。

GPT-5.3 Instant 是速度優化版本，在保持與 GPT-5.2 相近推理能力的同時降低延遲——類似 GPT-3.5 Turbo 與 GPT-4 的關係，以部分深度換取反應速度。這是驅動大多數即時應用場景的模型：客服聊天機器人、即時程式碼輔助、語音介面。

GPT-5.4 是面向消費者的頂級選項，提供 Standard、Thinking 和 Pro 三個版本。Thinking 版本在回應前會額外花時間逐步推理問題，是 OpenAI 將思維鏈推理從提示技巧提升為一級產品功能的實踐。Pro 版本則新增了擴展情境和專業工具調用能力。

以 GPT-4o 為基礎建立自訂 GPT 的企業客戶，在退役期間自動被遷移至最接近的 GPT-5.x 版本，OpenAI 提供了 90 天的緩衝期，供客戶在遷移正式生效前測試和調整自訂設定。

## 競爭現實：Gemini 在基準測試上佔優

OpenAI 的模型轉型，發生在比 GPT-4o 上市時競爭激烈得多的環境下。根據 Artificial Analysis 追蹤的 16 項主要 AI 基準測試，Google 的 Gemini 3.1 Pro 目前在其中 13 項領先，在 Artificial Analysis 智能指數上與 GPT-5.4 Pro 打成平手。

定價差距更為驚人。在同等任務上，Gemini 3.1 Pro 的 API 成本約為 GPT-5.4 Pro 的三分之一。對於運行大規模推論工作負載的企業客戶而言，這一差距的經濟意涵相當顯著——在高流量應用場景下，每年可轉化為數百萬美元的成本差異。

這個競爭背景，解釋了 OpenAI 在模型過渡上的部分緊迫性。公司無法承受客戶用還在供應中的 GPT-4o 去和 Gemini 3.1 Pro 做比較。GPT-5.4 Pro 在這個比較中是更有說服力的選項，即便它在部分基準上仍落後，且成本更高。

OpenAI 應對基準和定價挑戰的方式，是全力發揮生態系和整合優勢。公司的開發者工具——Assistants API、微調基礎設施，以及與外部服務的廣泛整合——對於深度建立在 OpenAI 平台上的組織而言，代表著真實的轉換成本。當你已部署了 50 個需要從頭重建的自訂 GPT，基準分數的高低就相對次要了。

## GPT-4o 退役對產業意味著什麼

GPT-4o 的退役，是 AI 模型格局移動速度的一個刻度。這個模型從發布到完全退役，約歷時 23 個月。相比之下，GPT-3.5 Turbo 作為 OpenAI 基準模型服役了近三年，才被大幅超越。

模型週期的加速，給企業客戶帶來了現實的挑戰。那些圍繞特定模型版本建立了合規、採購和安全審查流程的組織，發現自己面對的是持續不斷的變更管理，而非穩定的軟體部署。自訂 GPT 遷移的 90 天緩衝期雖有幫助，但與企業軟體團隊習慣的時程相比，仍相當壓縮。

這也為開源生態系帶來了有趣的動態。GPT-4o 上市時，相對於開源替代方案存在顯著的能力差距。到退役時，開源模型——尤其是 Meta 的 Llama 系列和 Mistral 的產品——在許多基準任務上已趕上甚至超越 GPT-4o。以合理成本即可自行部署的「免費層」AI 能力，在同一時期已大幅提升。

這意味著 OpenAI 的競爭護城河，愈來愈集中在前沿領域——需要最高深度推理任務的 GPT-5.4 Thinking/Pro 層級——而非過去曾是主要差異化來源的基準能力。廣義而言，這對企業 AI 採購者是更健康的市場結構：在商品層享有可信的免費和低成本替代方案，在需要時仍能取得前沿能力。

## 展望：GPT-4o 基礎設施還有一年壽命

對許多在 GPT-4o API 上建立應用程式的開發者而言，有一個重要的細節值得注意：ChatGPT 產品中的模型退役，不一定意味著 API 立即停用。OpenAI 一向維持棄用模型的 API 存取相當長的時間——通常 12 個月以上——以避免中斷已部署的應用程式。

截至本文撰寫時，GPT-4o 的 API 退役時程尚未公開說明，但根據慣例和 OpenAI 所承諾的開發者關係政策，API 存取很可能至少延續到 2027 年初。開發者應開始更新實作，但並非面臨緊急情況。

可以確定的是，GPT-4o 的時代已經終結。那個讓對話式 AI 對主流大眾真正展現能力的模型，已先被更好的模型取代，再被更好且更便宜的模型取代。這條軌跡，是 2024 至 2026 年 AI 能力發展故事的縮影。GPT-5.x 是當下。而它的繼任者，此刻已在某處悄然測試中。
