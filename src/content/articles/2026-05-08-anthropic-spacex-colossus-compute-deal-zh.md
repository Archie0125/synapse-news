在過去半年裡，Claude Code 的爆炸性成長讓 Anthropic 自有的基礎建設再也跟不上需求。5 月 6 日，這家以 AI 安全著稱的公司宣布了一個出人意料的解方：取得 SpaceX 旗下 Colossus 1 資料中心的全部算力。這座機房最初是為了運行 xAI 的 Grok 模型而建，如今卻要全力撐起 Claude 的用量高峰。這筆交易再次說明，在當前的 AI 軍備競賽中，算力的上限往往比模型本身更先觸頂。

## Colossus 1 帶來什麼

數字相當驚人。Colossus 1 讓 Anthropic 取得超過 30 萬瓦（300 MW）的電力容量，以及逾 22 萬張 Nvidia GPU，預計在公告後一個月內全數為 Anthropic 工作負載上線服務。這大約相當於一家中型超大規模雲端業者的專用 AI 訓練叢集的規模——而 Anthropic 過去一直以比競爭對手「更精實」的算力運作著稱。

協議以 SpaceX 為主體簽署，而非直接向 xAI 租借，反映出 SpaceX 作為 Colossus 1 實體設施擁有者與營運者的角色。協議還包含一個前瞻性的條款：Anthropic 表示已向 SpaceX「表達興趣」，探索在太空部署多吉瓦級別算力的可能性——這不像是臨時應急的語氣，更像是雙方關係延伸至未來的起點。

## 即時效益：Claude Code 限制鬆綁

Anthropic 動作迅速，第一時間將新增算力轉化為用戶可感知的改善。伴隨基礎建設公告，公司同步宣布 Claude Code 所有付費訂閱等級（Pro、Max、Team、Enterprise）的用量上限全面翻倍，並取消讓開發者深感困擾的尖峰時段使用限制。

取消尖峰時段上限尤其意義重大。Claude Code 正在經歷開發者工具中那種罕見的「病毒式採用」需求曲線：成長速度快到即便全球整體使用量只是略微偏高，特定地區的用戶在尖峰時段就會撞上限制。消除這道天花板，意味著 Anthropic 現在有足夠的算力餘裕，讓 Claude Code 更像一個隨時可用的基礎設施服務，而不是一個需要掐著用量的付費 API。

這對商業表現至關重要。Claude Code 的爆炸性成長是 Anthropic 營收加速的主要引擎，年化營收貢獻在 2026 年初已突破重要里程碑。每一次撞上用量上限，都是一次可能流失企業客戶的摩擦點；翻倍限制並取消尖峰管制，直接解決了最可能驅使企業客戶轉向競品的痛點。

## 用競爭對手的機房跑自己的模型

這筆交易最耐人尋味之處，在於其反直覺的結構。Anthropic 在大方向上是 xAI 的競爭對手——兩家公司都在競相打造前沿 AI 模型、爭奪企業客戶、搶佔開發者心智，以及角逐資本市場的青睞。Colossus 1 最初就是 SpaceX 為了跑 xAI Grok 的訓練任務而建造的。

如今 Anthropic 的工作負載卻跑在同一座機房裡，這提醒了整個行業：AI 基礎建設市場尚未沿著意識形態界線分割。算力是流動的，稀缺的是可用的容量。SpaceX 作為 Colossus 1 的所有者，有充分理由讓這座設施物盡其用，不論是哪家 AI 公司在付費。

背後也有結構性的商業邏輯：xAI 的 Grok 訓練排程存在高低起伏，Colossus 1 因此會出現閒置窗口。將這段閒置容量租給 Anthropic，讓 SpaceX 從原本可能空轉的基礎建設中獲得收益。對 Anthropic 而言，這解決了一個緊迫的供給問題，同時避開了自建同等規模基礎建設通常需要的 18 至 24 個月工期。

## 與 AWS 的合作疊加

SpaceX 的協議是 Anthropic 日趨多元化算力佈局中的最新一塊拼圖。2026 年 4 月，Anthropic 宣布擴大與 Amazon Web Services 的合作，共計最高 5 吉瓦的 AI 基礎建設容量，其中近 1 吉瓦的 AWS Trainium 2 算力預計於 2026 年底前上線。

多供應商策略有其清晰的戰略邏輯。AWS 提供 Trainium，幫助 Anthropic 在特定工作負載上降低對 Nvidia GPU 的依賴；SpaceX 的 Colossus 1 提供即時的 Nvidia GPU 容量，用於 Claude Code 等推論密集型服務；自有夥伴雲端基礎建設則服務特定產品線。

這種架構分散了集中於單一供應商的營運風險——過度依賴單一廠商在需求爆發時所帶來的脆弱性，已在其他 AI 公司的發展歷程中留下教訓。分散佈局也讓 Anthropic 在定價與供給談判中擁有更強的籌碼。

## 太空算力：遠大的想像空間

「在太空部署 AI 算力」這個說法很容易被歸入科幻類，但值得認真解讀。SpaceX 在這個方向上具有獨特的優勢：公司擁有全球最先進的商業發射能力，已部署數千顆 Starlink 衛星並在軌道上處理大量資料負載。

將 AI 算力移至太空，理論上可同時解決幾個重大瓶頸：地面資料中心的土地取得難題、龐大的能源消耗與碳足跡，以及透過低軌衛星路由推論流量以改善全球延遲。這些好處距離前沿 AI 所需的基礎建設規模仍有相當距離，但這次合作為認真探索這個方向創造了自然的 R&D 路徑。

對 Anthropic 而言，表態對太空算力感興趣，也傳遞了一個次要訊號：公司對基礎建設制約的應對視野，延伸至比當前算力擴充更長遠的時間軸。

## 這筆交易說明了什麼

Anthropic 與 SpaceX 的協議揭示了一個正在重塑 AI 公司基礎建設邏輯的市場現實：前沿 AI 競賽移動得如此之快，以至於任何公司——包括超大規模雲端業者——都無法純靠自建基礎建設趕上需求。結果是一個跨越傳統競爭界線的合作市場，一家公司的閒置容量成為另一家公司的策略資產。

對使用 Claude Code 的台灣開發者和企業而言，近期的直接感受是：拖累生產力的算力天花板已大幅提升，過去幾個月讓人頭疼的尖峰時段壅塞，應在數週內顯著舒緩。

更長遠的意涵則屬於結構性的。Anthropic 的年化營收已突破 440 億美元且持續加速，正在搭建一套與其野心相稱的基礎建設組合——自有容量、雲端巨頭夥伴、競爭對手機房租借，乃至軌道算力探索，一年前沒有人能預測到這樣的配置會成真。
