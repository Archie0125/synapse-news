Anthropic 對金融服務業發動了迅今最猛烈的攻勢。在紐約舉辦的閉門說明會上，公司發布了十款預製 AI 代理範本，專為銀行與金融業最耗時的工作流程量身打造；同步推出橫跨 Excel、PowerPoint、Word 與 Outlook 的 Microsoft 365 全面整合，讓 Claude 能在四個應用程式間保持連賫對話；以及與穆迪（Moody's）的里程碑合作，將全球逾 6 億家公司的信用與風險情報直接嵌入 Claude 介面。

這些公告伴隨 Claude Opus 4.7 的亮相——公司迅今錢對金融工作最強大的模型——標誌著 Anthropic 商業策略的決定性轉向。從過去販售 API 讓客戶自行構建，到現在直接出貨可立即部署的垂直工作流程解決方案，Anthropic 的目標是成為華物街的 AI 作業系統。

## 十款代理，今日上線

公告的核心是十款代理範本，每一款均設計用於自動化特定的高度勞力密集金融流程。五款面向研究與客戶覆蓋團隊：

- **Pitch Builder（簡報製作器）**：整合即時市場資料與客戶關係背景，自動撰寫投資建議書與簡報
- **Meeting Preparer（會議準備器）**：彙整新聞、財報與 CRM 歷史，為客戶電話生成會前簡報
- **Earnings Reviewer（財報審閱器）**：分析季度財報、提取關鍵指標，並標記與分析師共識的偏差
- **Model Builder（財務模型建構器）**：協助 Excel 財務建模，包括公式構建、情境分析與敦感度表格
- **Market Researcher（市場研究員）**：彙整並摘要產業趨勢、競爭對手情報與宏觀資料，支援投資論點

另外五款面向財務與營運團隊：

- **Valuation Reviewer（估值審核器）**：對比可比交易與市場基準，驗證估值假設的合理性
- **General Ledger Reconciler（總帳調節器）**：識別跨會計分錄的差異，標記需人工審查的項目
- **Month-End Closer（月結協調器）**：協調月末結帳流程，追蹤未完成事項並生成進度報告
- **Statement Auditor（財報審計員）**：審查財務報表的內部一致性、GAAP 合規性及異常狀況
- **KYC Screener（KYC 審查員）**：處理了解客戶（KYC）文件，標記風險指標，生成合規摘要

每款代理均以參考架構形式出貨——不只是提示詞，而是一套完整的技能、連接器、記憶系統與子代理的組合，足以端對端執行該工作流程。目標是讓金融機構在數天而非數月內部署出可運作的 AI 代理。

## Microsoft 365：跨四個應用的單一代理

Microsoft 365 整合代表了企業 AI 助理運作方式的質變。過去，大多數 AI 工具都是單一應用的點解決方案——Word 裡的寫作助理、Excel 裡的公式幫手、Outlook 裡的起草器。Claude 新的 M365 整合讓它成為一個在四個應用程式間保持完整上下文的單一持久代理。

銀行家在結構化一筆槓桿收購交易時，現在可以要求 Claude 從 Excel 提取財務模型、將更新後的假設整合進 PowerPoint 投影片、在 Word 備忻錄中摘要交易邏輯、在 Outlook 起草客戶郵件——整個過程是一場連賫的對話，Claude 在每個步驟間保持完整的上下文記憶。Excel、PowerPoint 與 Word 的外掛程式在發布活動上正式上線；Outlook 整合則進入測試版。

這套技術架構的意義值得關注。Claude 並非四個牃巧掛著微軟品牌的獨立 AI 整合，而是將 Microsoft 365 生態系當作一組可呼叫工具的統一推理層。這與 Anthropic 透過 Claude Code 及更廣泛代理 SDK 發展的代理架構一脈相承，現在被應用到了企業生產力軟體領域。

## 穆迪：6 億家公司的資料進入 Claude

穆迪合作是三項主要公告中策略意義最深遠的。穆迪將把其完整分析平台——包括全球逾 6 億家公司的信用評級、財務資料與風險評估——以原生應用程式形式嵌入 Claude。

對銀行或資產管理公司的信用分析師而言，這意味著請求 Claude 評估中市場借款人的信用狀況、與評級同業比較、識別契約風險，現在將得到以穆迪專有資料為基礎的答案，分析師無需離開 Claude 介面另開終端機或平台查詢。

穆迪合作是 Anthropic 更廣泛資料吸伴關係策略的一部分。Anthropic 正在打造企業資料整合市集——架構上確保吸伴容的專有資料能豐富 Claude 的回應，同時不被納入模型訓練或暴露給其他用戶。穆迪整合是迅今規模最大、資料最豐富的此類合作，涵蓋了一個主要信用情報平台的完整範疇。

## 雙軸市場攻略

Anthropic 的金融服務策略有兩條截然不同的路線。第一條针對一線機構——全球最大的銀行、資產管理公司與保險公司——提供工具與 API，讓它們在自身的安全與合規邊界內配置和部署 Claude 代理。這些機構通常希望在自己的雲端環境中運行 Claude，往往透過微軟 Azure 或亞馬遜雲端服務的企業協議。

第二條路線针對中市場：規模大到無法使用通用 SaaS 工具，但又缺乏高盛或貝萊德那種工程資源的公司。為這個客群，Anthropic 成立了一家私募股權支持的合資企業，將直接把 Claude 嵌入公司營運，提供代管服務模式而非自行配置方式。

雙軸策略反映了企業 AI 領域的一個普遂認識：最大的客戶要食材，中型客戶要成品。透過服務兩個客群，Anthropic 正在將 Claude 定位為金融 AI 的基礎設施層，無論機構的技術成熟度在哪個水準。

## 金融領域的 OpenAI 競爭

金融服務攻勢使 Anthropic 與 OpenAI 正面交锋——後者也一直透過與主要銀行的合作及其 Enterprise 產品線深耕金融垂直市場。

Anthropic 的差異化在於強調：OpenAI 著重讓通用模型更可配置，Anthropic 則直接出貨預製的垂直解決方案。十款代理範本代表了一個賭注：金融機構寵額部署一個經過驗證的工作流程，而不是從頭構建一個，即便後者提供更多彈性。

對台灣的金融業者而言，這波浪潮値得認真看待。從收集多個平台的資料、緜合分析到起草客戶文件，這些歷來需要分析師才能完成的工作，現在可由代理在幾分鐘內執行。AI 重塑金融業已是確定趨勢，問題只在於速度有多快，以及新堆疊中的哪個層次將捕獲最多價値。
