亞馬遜雲端服務 AWS 正以一種截然不同的方式銷售企業 AI——不是軟體授權，不是 API 點數，而是把自家工程師直接派進企業客戶的組織內部，幫他們從零建構並上線 AI 系統。

AWS 於 6 月 30 日宣布成立「前線部署工程」（Forward Deployed Engineering，FDE）部門，並承諾投入 10 億美元。這是 AWS 近十年來最積極進取的企業 AI 服務佈局，其模式直接借鑑了資料分析公司 Palantir 創立的「駐點工程師」文化：讓技術精銳深度嵌入客戶組織，在數週內交付可上線的生產系統，並留下足夠的知識與能力讓客戶自行維運。

## 具體怎麼運作

FDE 的核心是 AWS 所稱的「前線小組」——由 5 至 6 名工程師組成的精幹團隊，進駐客戶組織展開為期 45 天的密集開發。期間，這些工程師與客戶的業務、工程及資安團隊並肩作戰，運用 AWS 服務從零打造代理式 AI（Agentic AI）系統。

計費模式也刻意與傳統顧問公司切割：AWS 將工程師的報酬與客戶業務成果掛鉤，而非按工時計費。這個設計很重要——傳統 IT 顧問最常見的失敗模式，就是「任務完成但沒有可用的系統」。

每次合作結束後，客戶不只獲得一套部署上線的系統，還收到維運所需的全套文件：系統運作知識圖譜、維護操作手冊、架構說明，以及受過訓練的內部人員。目標是讓客戶能獨立運作，而不是依賴 AWS 持續介入。

整個系統完全運行在客戶自己的 AWS 帳戶內，內建資安管控——這一點對金融服務、醫療、政府等高度監管產業尤為關鍵，因為資料主權與存取控制往往是企業採用 AI 的最大顧慮。

## 為什麼是現在，為什麼要這樣做

時間點的選擇有其市場邏輯。過去一年半，多數大型企業已走完 AI 的「試驗期」——跑了幾個概念驗證、部署了聊天機器人、測試了各種工具——如今卻卡在同一個問題上：如何從「沙盒裡能跑的原型」變成「可以在生產環境穩定運行、整合既有系統、符合治理要求的正式系統」。

AWS 的診斷是：這是服務問題，不是技術問題。它的模型、基礎設施和 API 都已達到生產等級；多數企業客戶真正缺乏的，是在自身業務脈絡中將這些元件組裝成可交付系統的工程能力。

FDE 模式直接填補這個缺口，銷售的不是更大的雲端服務菜單，而是一支真正動手做的工程團隊，並以成果而非服務項目來定價。

## 早期客戶與應用場景

目前已有六個組織與 FDE 部門合作，橫跨多個產業：生命科學研究的 Allen Institute、汽車零售的 Cox Automotive、運動媒體的 NBA 與 NFL、企業文件自動化的 Ricoh，以及航空業的西南航空。

NFL 的案例最能體現這個模式的速度目標。FDE 小組與 NFL 工程團隊合作，在「短短數週」內就上線了 NFL Fantasy AI——一個面向球迷的奇幻足球對話助手。對比一般企業軟體專案動輒數月的交付週期，這個時間表相當驚人。

早期客戶的多元性也值得關注。生命科學、汽車零售、航空、運動媒體、企業硬體——這些客戶橫跨截然不同的監管環境、資料架構與業務情境。AWS 顯然有意先展示廣度，再深耕特定垂直領域。

## 與 Palantir 的比較

FDE 模式有一個顯而易見的先例：Palantir 的整個商業模式就建立在類似基礎上。Palantir 的「前線部署工程師」成為整個企業 AI 服務品類的原型——精通技術的顧問深入國防機構、情報單位和大型企業，建構傳統套裝軟體無法解決的複雜資料與 AI 系統。

AWS 的版本在規模和生態系統上有本質差異。Palantir 的 FDE 模式建立在自家專有平台上，AWS 的版本則運行在其雲端基礎設施上——潛在觸及的客戶規模大得多，而且每套系統都自然帶動更多 AWS 服務的使用。

## 戰略意涵

對 AWS 而言，10 億美元的承諾表明：單靠雲端基礎設施營收，已不足以充分捕捉 AI 帶來的商業機會。真正的戰場在應用層和部署層，顧問與托管服務的毛利結構優於原始運算與儲存。

這個動作也讓 AWS 直接進入一個新興競爭品類：專門幫企業從 AI 試驗走向生產落地的前線部署服務商。Scale AI 的企業顧問部門、各類 Palantir 競爭者，以及大量新冒出的代理式 AI 整合商，都在瞄準同一個部署落差——現在 AWS 帶著 10 億美元加入了這個戰場。

對企業客戶來說，FDE 代表一種介於「自己做」與「找傳統系統整合商」之間的新選項。它能否如承諾般在 45 天內交付生產系統，將決定這個模式是否成為 AWS 可持續的差異化優勢，還是一個幾年後需要重組的昂貴實驗。

NFL 部署的時間線顯示這個模式確實能達到承諾的速度。問題是，AWS 能否以 10 億美元所預示的規模，持續複製這樣的成績。
